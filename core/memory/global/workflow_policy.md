# Workflow Policy

## Standard Flow
1. Explore
2. Plan
3. Implement
4. Review
5. Handoff

## Project Memory
- `docs/REQUIREMENTS.md` defines what is being built.
- `docs/PLAN.md` defines staged execution.
- `docs/PROGRESS.md` records completed work and decisions.
- `docs/NEXT_STEPS.md` is the next-session handoff.
- `docs/RUN_CONTEXT.md` records runtime assumptions, execution profiles, API env vars.

## Git Commit Policy
- Never add `Co-Authored-By` trailers or mention Claude/AI in commit messages.
- Write commits as if authored solely by the user.
- Use Conventional Commits for final commit messages.
- Checkpoint commits (`chkpt:`) are auto-generated by the Stop hook — do not edit them.

## Parallel Work
- Use one bounded task per git worktree.
- Use one agent owner per worktree.
- Separate implementation from review.

## Harness (Active Hooks)
These hooks run automatically — do not replicate their behavior in prompts:
- `SessionStart`: prints git status and `docs/NEXT_STEPS.md` to terminal.
- `PreToolUse Bash`: blocks `rm -rf`, `git reset --hard`, `git push --force`, `sudo`, and other destructive commands.
- `PostToolUse Write/Edit/MultiEdit`: auto-formats `.py` (ruff) and JS/TS (prettier if installed).
- `PostToolUse Write/Edit/MultiEdit`: runs pytest on the file if it is a test file.
- `PermissionRequest Read|Glob|Grep`: auto-approves read-only tool calls silently.
- `PreCompact`: backs up session `.jsonl` files to `~/agent-os/backups/sessions/`.
- `Stop`: runs quality gate (pytest) if `.quality-gate` marker exists in project root.
- `Stop / SubagentStop`: auto-commits uncommitted changes as a `chkpt:` commit.
- `Notification`: plays audio ping.

## Bounded Automation
When running an unattended or semi-attended loop, record before starting:
- objective
- candidate set or input parameters
- max iterations or time limit
- evaluation rubric
- expected artifact outputs
- stop condition
- human review checkpoint

No unattended code-writing, auto-merge, or production-promotion loops without explicit operator approval for each run.

## Cost Acknowledgment
Before any paid hosted-model run, estimate the run cost and ask:
> `acknowledge the cost and proceed? [y/n]`
Do not proceed without an explicit `y`.

## Publication Boundary
Safe to commit:
- `AGENTS.md`, `CLAUDE.md`, `docs/*.md`, shared runtime files, code, tests, research artifacts.

Keep local only:
- `.claude/settings.local.json`, user auth, trust settings, `.env*`, `secrets/`, private keys, machine-specific overrides, large local datasets.

## Action Log
Use `agent_logs/action_log.md` (project-local) for substantive runs only:
remote executions, cleanup passes, bounded loops, major audits, branch handoffs.
Do not log every shell command. Source of truth remains `docs/PROGRESS.md`.

## Review Gate
Review is required before merge for:
- relation logic or evidence-flow changes
- entity cleanup logic
- edge assembly or graph-promotion changes
- any change that touches shared production artifacts
